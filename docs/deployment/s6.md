# s6 / s6-rc

Non-systemd deployment for Adelie, Artix-s6, or any host using
s6 as the supervisor.

## Heads-up: the `run` file is execlineb, not sh

s6's idiom is to use `execlineb` (a non-shell language) for
service definitions. `deploy/init/s6/shellboto/run` starts with:

```
#!/bin/execlineb -P
```

This means shellcheck can't lint it (it's not a shell script).
We deliberately exclude it from shellcheck hooks; ran through
s6's `s6-rc-compile` instead.

## Install

Assuming an s6-rc source tree at `/etc/s6-rc/source`:

```bash
sudo ./deploy/install.sh --skip-systemd

sudo mkdir -p /etc/s6-rc/source/shellboto
sudo install -m 0755 deploy/init/s6/shellboto/run /etc/s6-rc/source/shellboto/run
sudo install -m 0644 deploy/init/s6/shellboto/type /etc/s6-rc/source/shellboto/type

sudo s6-rc-compile /etc/s6-rc/compiled /etc/s6-rc/source
sudo s6-rc-update /etc/s6-rc/compiled
sudo s6-rc -u change shellboto
```

## The `run` file

```
#!/bin/execlineb -P

# Redirect stderr to stdout so s6-log catches both streams.
fdmove -c 2 1

# Load env vars; s6-envdir expects one-file-per-var but we keep a single
# env file for parity with systemd — shim through a trivial bash loader.
execlineb -c {
    importas -i HOME HOME
    /bin/sh -c "set -a && . /etc/shellboto/env && exec /usr/local/bin/shellboto -config /etc/shellboto/config.toml"
}
```

- **`fdmove -c 2 1`** — redirect stderr to stdout.
- The nested `execlineb -c { ... }` block falls back to `/bin/sh`
  to source the env file. s6-envdir (one file per variable) is
  the canonical s6 way, but it's a pain to manage alongside the
  systemd env file — so we shim.
- **`exec`** replaces the shim shell with shellboto.

## The `type` file

```
longrun
```

Tells s6-rc this service is a long-running daemon (supervise +
restart on exit).

## Logging

Pair with `s6-log`:

```
#!/bin/execlineb -P

# /etc/s6-rc/source/shellboto-log/run
s6-log -d3 -- t /var/log/shellboto
```

`t` prefixes each line with TAI timestamps. Similar to svlogd.

## Differences from runit

- **execlineb semantics**, not POSIX sh. Scripts look strange but
  compose better for supervised services.
- **s6-rc is a dependency graph.** You can express "shellboto
  wants network" declaratively.
- **More moving parts** than runit. Higher ceiling, steeper ramp.

## Starting / stopping

```bash
sudo s6-rc -u change shellboto     # up
sudo s6-rc -d change shellboto     # down
sudo s6-svstat /run/service/shellboto
```
